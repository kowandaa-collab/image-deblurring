Introduction
Background

Image deblurring restores blurry images to sharp, clear images — a fundamental low-level vision task with broad real-world impact.

Causes of Blur
  • Camera shake during long exposure
  • Fast-moving objects (motion blur)
  • Low-light environments requiring slow shutter
Applications
  • Photography & Surveillance
  • Medical Imaging
  • Autonomous Driving

Problem Statement
Problem

Blurred images lose important visual information, making restoration an ill-posed inverse problem — one blurred image may correspond to infinitely many sharp images.

Challenges
  • Recovering fine details destroyed by blur
  • Preserving edge structure and texture
  • Handling spatially-varying blur kernels
  • Avoiding ringing and over-sharpening artifacts
Research Question
Can image deblurring performance be improved by injecting BlurDM diffusion priors into a NAFNet-based restoration backbone via Feature-wise Linear Modulation (FiLM)?

BlurDM Overview

Unlike standard diffusion that only adds Gaussian noise, BlurDM simultaneously introduces blur + noise during the forward process — modeling the physical degradation directly.

Key Properties
  • Dual forward process: blur residual + noise residual
  • Separates noise and blur in latent space
  • Reverse process reconstructs sharp image step-by-step
  • Uses Blur Encoder to extract degradation latent zB

    y = self.norm(x)
    f = self.freq(y)          # FFT global branch — captures motion blur frequency signature

Proposed Method — FiLM-Conditioned Backbone
FiLM Injection Formula
FiLM(x) = γ · x + β
[γ, β] = Linear(p)
init: γ=1, β=0 (identity)
HybridBlurDM.py — FiLMInjection 
class FiLMInjection(nn.Module):
    def __init__(self, channels, prior_dim):
        self.film = nn.Linear(prior_dim, channels * 2)
        nn.init.ones_(self.film.bias[:channels])   # γ=1
        nn.init.zeros_(self.film.bias[channels:])  # β=0

    def forward(self, x, prior):
        film = self.film(prior).unsqueeze(-1).unsqueeze(-1)
        scale, shift = film.chunk(2, dim=1)
        return x * scale + shift  # FiLM(x) = γ·x + β

def fft_loss(criterion, pred, target):
    pf = torch.fft.rfft2(pred,   norm="backward")
    tf = torch.fft.rfft2(target, norm="backward")
    return criterion(
        torch.stack([pf.real, pf.imag], dim=-1),
        torch.stack([tf.real, tf.imag], dim=-1),
    )

Discussion
Strengths
  • Effective blur removal across diverse real-world motion blur scenes
  • Better feature extraction — DualDomainMixer captures both spatial and frequency patterns simultaneously
  • HybridBlurDM achieves 31.96 dB surpassing NAFNetBlurDM (31.71 dB) even without the DM prior
Limitations & Why HybridBlurDM Is Behind
  • High computational cost — FFT operations and deformable attention are significantly more expensive than standard convolutions
  • Long training time — 300+ epochs per stage required for convergence
  • Incomplete 3-stage pipeline — HybridBlurDM only completed standalone training; Stage 2 (DM) and Stage 3 (joint fine-tune) were not run
  • Time lost to model errors — repeated architectural bugs in DualDomainMixer, MotionStripConv & deformable attention delayed the full pipeline
